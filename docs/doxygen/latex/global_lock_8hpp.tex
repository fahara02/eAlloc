\doxysubsection{Detailed Description}
Universal RAII lock guard and platform lock adapter system for e\+Alloc. 

Provides MCU-\/agnostic, minimal-\/overhead locking for critical sections and memory allocators. Selects the appropriate mutex adapter for each platform via CMake and preprocessor macros.

Usage\+:
\begin{DoxyItemize}
\item Use \doxylink{classglock_1_1_i_lockable}{glock\+::\+ILockable} as the abstract mutex interface.
\item Use \doxylink{classglock_1_1_lock_guard}{glock\+::\+Lock\+Guard} for RAII critical sections.
\item Use the correct adapter (e.\+g. glock\+::\+Free\+RTOSMutex, glock\+::\+Std\+Mutex) for your platform.
\item Platform is selected by CMake option and macro (EALLOC\+\_\+\+PC\+\_\+\+HOST, FREERTOS, etc).
\end{DoxyItemize}

Thread Safety\+:
\begin{DoxyItemize}
\item Only one lock per memory pool/allocator instance is recommended.
\item Avoid double-\/locking by letting the allocator own the lock and exposing set\+Lock().
\end{DoxyItemize}

No STL bloat for MCU\+: STL headers only included for PC/host builds.
